# برنامه مرحله‌ای بهبود پروژه (بدون تغییر API)

این برنامه دقیقاً مطابق خواسته شما طراحی شده: **هیچ تغییری در API انجام نمی‌دهیم** و مرحله‌به‌مرحله روی کیفیت محصول جلو می‌رویم.

## فاز 1 — بهبود مستندات و شفافیت پروژه (شروع فوری)

### اهداف
- کاهش ابهام برای مشارکت‌کننده جدید
- کوتاه کردن زمان Onboarding

### کارها
1. تکمیل README (انجام شد).
2. اضافه کردن بخش «ساختار ریپو» و «سناریوهای رایج استفاده» در Wiki یا docs.
3. تعریف یک چک‌لیست ساده برای PRها (توضیح، تست، ریسک).

### خروجی قابل تحویل
- مستندات یکپارچه و قابل‌فهم برای همه اعضا.

---

## فاز 2 — ماژولار کردن UI و بهبود نگه‌داری

### اهداف
- کاهش پیچیدگی فایل‌های بزرگ
- افزایش خوانایی و تست‌پذیری

### کارها
1. شکستن `app.html` به کامپوننت‌های کوچک:
   - `app-header`
   - `agent-form`
   - `preview-panel`
   - `export-actions`
2. کاهش مسئولیت `App` و انتقال بخشی از منطق UI به لایه‌های جدا.
3. تعریف typeهای دقیق‌تر به‌جای `any` در مسیر داده فرم.

### خروجی قابل تحویل
- ساختار کامپوننتی تمیزتر با هزینه نگه‌داری کمتر.

---

## فاز 3 — افزایش کیفیت خروجی و امنیت سمت کلاینت (بدون تغییر API)

### اهداف
- خروجی امن‌تر و قابل‌اعتمادتر
- کاهش ریسک محتوای ناخواسته در export

### کارها
1. افزودن sanitize برای HTML قبل از دانلود HTML/PDF.
2. اعتبارسنجی بهتر داده‌های ورودی فرم (طول، قالب لینک‌ها، فیلدهای کلیدی).
3. بهبود مدیریت خطاها با پیام‌های دقیق‌تر برای کاربر نهایی.

### خروجی قابل تحویل
- کیفیت بهتر خروجی‌ها با ریسک کمتر در فایل‌های export شده.

---

## فاز 4 — تست و پایداری

### اهداف
- جلوگیری از Regression
- افزایش اطمینان در توسعه سریع

### کارها
1. افزودن unit test برای:
   - `LanguageService`
   - `ThemeService`
   - `GeminiService` (در سطح رفتار و map خطا)
2. افزودن integration test برای flow اصلی:
   - پر کردن فرم
   - تولید خروجی
   - نمایش خطا
3. تعریف حداقل پوشش تست هدف (مثلاً 60٪ شروع).

### خروجی قابل تحویل
- baseline تست قابل اتکا برای توسعه‌های بعدی.

---

## فاز 5 — Polish نهایی تجربه کاربری

### اهداف
- تجربه کاربری حرفه‌ای‌تر
- دسترسی‌پذیری بهتر

### کارها
1. بهبود Accessibility:
   - label/aria کامل‌تر
   - کنتراست رنگ
   - focus state مناسب
2. بهبود UX متن‌های خالی و empty states.
3. بهینه‌سازی micro-interactionها بدون پیچیده کردن کد.

### خروجی قابل تحویل
- UI روان‌تر، دسترس‌پذیرتر، و آماده استفاده عمومی.
